## 未解決の問題リスト

### 重要度高（機能的問題）

1. **球面投影の半球クリッピングで地平線弧が欠落している**
    - `clipPolygonToHemisphere` / `clipPolygonVectors` が多角形を z >= 0 の半球に切り取る際、切断点間を**直線**で結んでおり、球の輪郭（単位円弧）に沿った弧セグメントを追加していない
    - 結果: 球の側面付近で塗りつぶし領域が「途切れて」見え、球の端まで正しく延びない（弦と弧の間の面積が欠落する）
    - 正面は全頂点が半球内にあるので問題ないが、球端に近い多角形ほど顕著に歪む
    - 陸マスク・選択領域マスク・領海マスクすべてに波及するため、側面の描画全体が崩れる
    - 修正方針: 切断点で半球を出入りする際、出口→入口間に地平線弧（z=0 の単位円弧）を補間点で追加する

2. **欠番**

3. **`getWikiSize` の上限値とデフォルト値の矛盾**
    - HTMLの `<input id="wikiSize">` のデフォルトは `4800` だが、`getWikiSize()` (665行目) が `Math.min(2000, ...)` で 2000 に制限している
    - ユーザーが4800を入力しても実際には2000pxで出力されるため、見た目上の設定と実際の出力が乖離
    - コミット `feat: wiki風サイズのデフォルトを10倍` でデフォルト値だけ変更し、上限値の更新を忘れたと推定
    - 修正方針: `Math.min` の上限を4800以上に変更するか、HTML側のデフォルトを2000にする

    - トポロジーが変わるので扱い難しいかも
    - 穴の中の飛び地の中の穴……には対応しなくてよい
    - 現状、外側の国を選択した状態だと包領までハイライトされてしまうが、包領をShift選択すると包領だけ暗く出来る。

### 重要度最低（未顕在の不具合）

1. **`splitPathData` が logic.js と index.html で実装が異なる**
    - logic.js (15行目): `/(?=M)/` — 大文字 `M` のみで分割
    - index.html (697行目): `/(?=[Mm])/` — 大文字小文字両方で分割
    - 小文字 `m`（相対 moveto）を含む SVG パスデータで動作が不一致になる可能性
    - logic.js 側が古いまま残っている

2. **logic.js と index.html の間で多数の関数が重複定義されている**
    - `splitPathData`, `applyTransform`, `samplePathElement`, `samplePolylineElement`, `distance` / `distanceSq` など、同じ目的の関数が両方に存在
    - 片方だけ修正すると不一致バグが生まれる（splitPathData が実例）
    - logic.js は `DerleLogic` モジュールとして公開しているが、index.html 側が独自にローカル関数を再定義しており、どちらが使われるか分かりにくい

3. **`polygonSurface` が設定されるが使用されていない**
    - `buildWikipediaCache()` で `state.wikiOutput.polygonSurface` に海/陸の分類を格納するが、`buildWikipediaSvgMarkup()` では一切参照されていない（デッドコード）

4. **logic_test.html が存在しない**
    - 要件定義書 §4 に「ロジック検証用の HTML（logic_test.html）」の記載があるが、リポジトリにファイルが存在しない

### 設計・開発工程の問題

    - 関心の分離ができておらず、保守・レビュー・デバッグが困難
    - logic.js との分離も中途半端で、責務の境界が曖昧
